# Milestone v1.1: Accrue Seam Hardening

**Status:** ✅ SHIPPED 2026-04-29
**Phases:** 6-7
**Total Plans:** 3

## Overview

Closed the consumer-contract gaps Accrue needed to confidently consume oarlock as its Paddle backend. Added the missing `Paddle.Transactions.get/2` retrieval surface (TXN-03), locked the end-to-end Accrue seam with an adapter-backed contract test (SEAM-01), and published a canonical seam guide with sealed docs surface (SEAM-02). The seam now uses locked/additive/opaque vocabulary, hides internal modules from generated docs, and treats `:raw_data` as a presence-only escape hatch on every struct.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- Locked field-tier vocabulary across the seam guide: `locked`, `additive`, `opaque` — replacing earlier `raw` / `not-planned` terminology. `:raw_data` is `locked` on every struct row with `opaque` contents.
- Closed-enumeration boundary: only documented modules, functions, structs, and support types are supported; undocumented internals may change without notice in 0.x.
- Hid `Paddle.Http`, `Paddle.Http.Telemetry`, and the placeholder root `Paddle` module from generated docs via `@moduledoc false`.
- Seam test freezes only documented locked guarantees from `guides/accrue-seam.md`; six payload-equality assertions replaced with `is_map/1` escape-hatch presence checks.
- Seam test cross-step continuation flows through the locked typed seam (`fetched_transaction.subscription_id`) instead of opaque `event.data["subscription_id"]`.

**Issues Resolved:**

- TXN-03 retrieval gap closed; Accrue's checkout reconciliation flow unblocked.
- End-to-end Accrue journey now has a single deterministic, offline contract test (`test/paddle/seam_test.exs`).
- Consumer-facing contract is published and renderable as `guides/accrue-seam.md` + ExDoc.

**Issues Deferred:**

- Subscription mutations (`update`, `pause`, `resume`) — explicitly out of v0.1; Accrue does not depend on them.
- Payment-method update flows — out of v0.1.
- Refunds, invoice generation, marketplaces — out of v0.1.

**Technical Debt Incurred:**

The summaries have been annotated with retro notes (`docs(retro): correct SUMMARY drift…`, commit `4470053`) for audit honesty. Future phase execution should run `git status` before writing the SUMMARY to prevent this drift.
