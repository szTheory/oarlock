## v1 Requirements (carried forward, all validated)

### Core SDK
- [x] **CORE-01**: Explicit client instantiation (`Paddle.Client.new!/1`) with Bearer auth, base URLs, and API version header ("1"). *(Validated in Phase 1.)*
- [x] **CORE-02**: HTTP transport built on `req` with built-in JSON parsing, retries, and telemetry. *(Validated in Phase 1.)*
- [x] **CORE-03**: Consistent typed responses: `{:ok, struct}` and `{:error, %Paddle.Error{}}`. *(Validated in Phase 1; applied throughout.)*
- [x] **CORE-04**: Retain raw response payloads (e.g., `raw_data`) in structs for forward compatibility. *(Validated in Phase 1; applied throughout.)*
- [x] **CORE-05**: Pagination support returning `{:ok, %Paddle.Page{data: [...], meta: ...}}`. *(Validated in Phase 1; exercised in Phase 3 list flows.)*

### Webhooks
- [x] **WEB-01**: Pure function signature verification (`Paddle.Webhooks.verify_signature/4`). *(Validated in Phase 2.)*
- [x] **WEB-02**: Support configurable timestamp tolerance (default 5s) and multiple `h1` signatures. *(Validated in Phase 2.)*
- [x] **WEB-03**: Event parsing into typed structs (`Paddle.Webhooks.parse_event/1`). *(Validated in Phase 2.)*

### Customers & Addresses
- [x] **CUST-01**: Map application users to Paddle Customers (create, get, update). *(Validated in Phase 3.)*
- [x] **ADDR-01**: Support customer billing addresses (create, list, update). *(Validated in Phase 3.)*

### Transactions
- [x] **TXN-01**: Create recurring transactions mapping to hosted checkouts. *(Validated in Phase 4.)*
- [x] **TXN-02**: Return hosted checkout URLs from transaction creation. *(Validated in Phase 4.)*

### Subscriptions
- [x] **SUB-01**: Fetch canonical subscription state from Paddle. *(Validated in Phase 5.)*
- [x] **SUB-02**: List subscriptions for a customer. *(Validated in Phase 5.)*
- [x] **SUB-03**: Cancel subscription with end-of-period and immediate cancellation semantics. *(Validated in Phase 5 — `cancel/2` and `cancel_immediately/2` with hydrated `%ScheduledChange{}` and `%ManagementUrls{}`.)*

## Out of Scope (reaffirmed at v1.1 close)

- **Paddle Classic Support**: Legacy API concepts and authentication (vendor-id).
- **Phoenix/Ecto coupling**: No framework dependencies, UI dashboards, migrations, or schemas in core.
- **Payment Method Portals**: Excluded for v0.1.
- **Invoice Generation**: Excluded for v0.1.
- **Refunds**: Excluded for v0.1.
- **Marketplaces / Connect**: Excluded for v0.1.
- **Subscription mutations** (`update`, `pause`, `resume`): Not on near-term roadmap; Accrue's first slice does not depend on them.
