<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw webhook body -> `parse_event/1` | Untrusted external JSON enters the SDK and must be decoded without assuming required keys exist. |
| decoded payload -> `%Paddle.Event{}` | Only known envelope fields should be promoted while the original payload remains preserved in `raw_data`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | `Paddle.Webhooks.parse_event/1` | mitigate | Decode with `Jason.decode/1`, reject invalid JSON with explicit error tuples, and require the exact top-level keys before building `%Paddle.Event{}`. |
| T-02-02 | I | `Paddle.Event.raw_data` | accept | Preserve the full decoded webhook payload intentionally for forward compatibility; the SDK does not log or transmit it in this phase. |
| T-02-03 | D | `Paddle.Webhooks.parse_event/1` | mitigate | Keep parsing synchronous and local only, with no network calls or dynamic dispatch on event type. |
| T-02-04 | R | `test/paddle/event_test.exs` | mitigate | Lock the public tuple and struct contract with exact ExUnit assertions so silent payload-shape drift becomes a test failure. |
</threat_model>
