| Paddle checkout payload -> `%Paddle.Transaction.Checkout{}` | The nested checkout payload must become a dot-accessible Elixir shape instead of leaking a string-key map into the public API. |

| T-04-02 | I | `raw_data` preservation | accept | Preserve the full upstream payload intentionally in `raw_data` for forward compatibility while keeping promoted fields explicit and small. |
| T-04-03 | T | nested checkout mapping | mitigate | Use a dedicated `%Paddle.Transaction.Checkout{}` contract so the public API cannot leak a plain string-key checkout map that breaks `transaction.checkout.url`. |
| T-04-04 | R | entity contract stability | mitigate | Freeze the struct field surfaces and raw-data behavior in `test/paddle/transaction_test.exs` before `Paddle.Transactions.create/2` is implemented. |
